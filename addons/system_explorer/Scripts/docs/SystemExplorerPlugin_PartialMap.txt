System Explorer partial class map:

SystemExplorerPlugin.cs (Core file)
- Constants, shared fields, editor lifecycle, icon loading, template bootstrap, dock registration/removal, and core setup/teardown flow.
- This is the only partial file that carries [Tool] and inherits from EditorPlugin.

SystemExplorerPlugin.DebugLogging.cs
- Contains debug-only logging helpers guarded by DebugState.
- Provides DebugLog(...), DebugLogOperation(...), DebugLogStateSnapshot(...), and DebugLogSystems(...).
- Used by other partial files for developer diagnostics without noisy normal-user output.

SystemExplorerPlugin.DockUi.cs
- Builds the dock UI controls, toolbar/input layout, dialogs, tree control, context menu nodes, and context menu signal wiring.
- Handles System Name input behavior, including the inline Enter hint icon and cursor updates.

SystemExplorerPlugin.TreeBuilding.cs
- Builds/rebuilds the tree, restores expansion state, tracks forced expansion, and contains collapse/expand helpers.
- Provides tree expansion snapshot helpers used by operations that must restore the user's exact visual tree state after temporary editor/script sync work.
- Handles lock icon visibility/display state and script tooltip text during tree creation.

SystemExplorerPlugin.ScriptFilter.cs
- Handles script filtering, filter result tree generation, filter UI icons, clear/search icon behavior, and restoring the normal tree after filtering.
- Handles revealing/selecting a script again after exiting filter mode.

SystemExplorerPlugin.AddCreate.cs
- Handles adding systems/folders/scripts/scenes and creating new C# script files from the template.
- Handles selected file callbacks, target location resolution, forced expansion after additions, and opening newly created scripts through the shared System Explorer script-open helper.

SystemExplorerPlugin.NavigationRecovery.cs
- Handles tree item selection/navigation for scripts and scenes, reopening selected scripts, and missing script recovery dialogs.
- Handles relinking or removing missing scripts and safely replacing entries in systems.
- Delegates System Explorer script opening to SystemExplorerPlugin.ScriptOpenPosition.cs so navigation behavior stays centralized.

SystemExplorerPlugin.ScriptOpenPosition.cs
- Contains the shared System Explorer script-open helper used by tree navigation and newly created scripts.
- Uses EditorIntegration/ScriptEditing/ScriptPathUtility.cs for general script-path string normalization.
- Tracks which script paths have been opened from System Explorer during the current editor session.
- After the first System Explorer open for a script in a session, verifies the active script editor and attempts to place the TextEdit caret/scroll position at the top.
- Releases tree focus after navigation when requested and logs skipped/failed top-reset attempts through DebugLog(...).

SystemExplorerPlugin.ScriptEditorBuffers.cs
- Contains shared compatibility wrappers and orchestration for working with Godot’s open script editor buffers and script text files.
- Uses EditorIntegration/ScriptEditing/ScriptPathUtility.cs for general script-path string normalization.
- Keeps thin compatibility wrappers for general text-file reading/writing plus disk-verification comparison/normalization, delegating their implementation to EditorIntegration/ScriptEditing/ScriptTextFileService.cs.
- Keeps compatibility wrappers for unsaved-buffer checks and committed text application, delegating TextEdit saved-state transitions to EditorIntegration/ScriptEditing/ScriptEditorBufferStateService.cs.
- Retains the existing dictionary iteration, path matching, missing-entry handling, and newline result formatting used when checking or applying multiple open buffers.
- Retains activating open-buffer lookup and deferred path-payload build/parse orchestration.
- Keeps the TryAutosaveOpenScriptEditorBufferIfNeeded(...) compatibility wrapper, operation-specific autosave failure-message formatting, and TryAutosaveOpenScriptEditorBuffersIfNeeded(...) dictionary iteration while delegating general per-buffer autosave and disk verification to EditorIntegration/ScriptEditing/ScriptEditorBufferAutosaveService.cs.
- Keeps Beautify's specialized autosave workflow separate because it returns original text and participates in editor-first application and failed-write restoration.
- Keeps thin compatibility wrappers for non-activating lookup and delegates the matching implementation to EditorIntegration/ScriptEditing/ScriptEditorBufferLocator.cs.
- Keeps Refactor Namespace-specific unmatched-using protection, operation-specific lookup failure messages, payload handling, and the remaining shared buffer orchestration used by Refactor Namespace and Beautify.
- Applies committed text across multiple open editor buffers through ScriptEditorBufferStateService while preserving the existing operation-specific dictionary behavior.
- Godot resource refresh is owned by EditorIntegration/ScriptEditing/ScriptResourceRefreshService.cs; general per-buffer autosave and verified saved-version transition are owned by ScriptEditorBufferAutosaveService, while operation-specific messages, batch iteration, activating lookup, deferred payload handling, and editor/view-state responsibilities remain in the plugin partials.

SystemExplorerPlugin.ScriptEditorSync.cs
- Handles synchronization between Godot's active Script Editor script and the System Explorer tree selection.
- Keeps the specialized NormalizeScriptPathForSync(...) wrapper, using ScriptPathUtility.Normalize(...) before and after ProjectSettings.LocalizePath(...) while retaining its res:// and .cs validation.
- Connects to ScriptEditor active-script changes when available, with a light polling fallback if the editor signal cannot be connected.
- Primes the initially active editor script without changing tree selection, then syncs only after subsequent active-script changes or explicit post-operation requests.
- Provides follow-active-script behavior and post-operation selection resync helpers for stabilizing tree selection after single-script refactor/format operations.
- Provides a short-lived active-script suppression scope so internal ScriptEditor activations during batch/editor-buffer operations do not leak into user-facing tree selection or expansion.
- Provides a separate batch context-preservation scope that suppresses post-operation active-script sync until a deferred pass, allowing already queued post-operation sync calls to no-op after batch operations.
- Preserves existing expansion state additively by only expanding the minimum parent path needed to reveal the active script.
- Does not mutate system data or persist systems; only updates visual tree selection/expansion defensively.

SystemExplorerPlugin.Shortcuts.cs
- Handles global plugin shortcut routing through _Input(...), including Ctrl+L lock, Ctrl+B Beautify, Ctrl+Delete remove, and Ctrl+Shift collapse detection helpers.
- Uses ScriptPathUtility.Normalize(...) for the active Script Editor script path before routing Beautify.
- Routes selected-item shortcut actions such as Beautify and remove without changing the underlying tree mutation safeguards.
- Allows only Ctrl+B / Beautify to pass through when focus is safely verified inside the active Godot script editor, so the active editor script can be formatted without falling back to the selected System Explorer tree item.

SystemExplorerPlugin.TreeInput.cs
- Handles Tree.GuiInput and mouse-exit behavior for tree keyboard/mouse input, hover lock icon updates, filtering clicks, double-click scene opening, drag preparation/release, and keyboard collapse routing.
- Contains tree-item input helpers for script/scene detection, selected script/scene click handling, shift-click expansion, and local expand/collapse toggling.

SystemExplorerPlugin.ContextMenu.cs

- Captures the exact selected script's system/folder/entry, filter, expansion, and pre-dialog focus context before the context menu takes focus so script rename can restore the same location after rebuilding.
- Delegates physical file/folder target resolution and OS file-manager opening to SystemExplorerPlugin.ContextMenu.FileManager.cs.

SystemExplorerPlugin.ContextMenu.FileManager.cs

SystemExplorerPlugin.DialogInput.cs

SystemExplorerPlugin.ItemLocking.cs

SystemExplorerPlugin.SceneLinking.cs

SystemExplorerPlugin.DragDrop.cs

SystemExplorerPlugin.RenameRemove.cs
- Handles rename and remove operations for systems, folders, scripts, linked scenes, and direct scene entries.
- Handles file deletion, file/folder renames, forced expansion after renames, and updating script/scene entries after path changes.
- Physical remove now validates all target paths before mutation, safely autosaves and closes matching open C# Script Editor tabs through ScriptEditorTabService, verifies that Godot released the old Script resources, deletes files and UID sidecars with explicit result reporting, and only then mutates/persists _systems and performs one final EditorFileSystem scan.
- Script rename begins from the already-open script selected through System Explorer, captures exact Script/TextEdit instance IDs, saved/unsaved buffer text, caret, horizontal/vertical scroll, selection, active-script state, exact system/folder/entry identity, filter selection, expansion state, and intended tree focus.
- Reuses the existing autosave I/O and disk-verification policy before close; autosave failure leaves the tab, filesystem, _systems, JSON, tree selection, and filesystem scan untouched.
- Detects case-insensitive duplicate open Script resources before mutation, lists their actual ResourcePath values, and cancels rather than choosing an arbitrary resource.
- Uses EditorIntegration/ScriptEditing/ScriptEditorTabService.cs under ScriptEditorSync suppression to close the exact old Script/TextEdit pair through Godot 4.6's own Script Editor close lifecycle, then verifies through GetOpenScripts() that neither the old instance nor old path remains open before filesystem mutation.
- Supports normal one-step rename and case-only rename through a unique temporary sibling .cs path, with rollback to the original path when the second step fails; recovery of the unchanged original script after rename or close failure uses the same deferred reopen/verification flow as a successful rename.
- After verified filesystem success, refreshes Godot once, mutates and saves _systems defensively, rebuilds the tree with preserved expansion, and then treats EditorInterface.EditScript(...) only as a request to open the freshly cache-ignored Script resource from the final path.
- Owns a single operation-specific PendingScriptRenameEditorRestore state at a time. A new script rename is blocked while that state is active, and ScriptEditorSync suppression remains active until the pending editor and final tree restore sequence completes or is cancelled during plugin shutdown.
- Verifies reopen on up to three separate deferred passes. Temporary states such as a missing current editor/TextEdit, a not-yet-current unique final Script, or a not-yet-listed final Script are retried without an unbounded queue; definitive ambiguity, missing-file, old-instance, old-path, duplicate-final-path, casing, or buffer-text failures stop immediately.
- Uses the exact normalized final path plus the unique Script resource exposed through GetOpenScripts() as the source of truth. The current Script does not have to be the exact Script instance originally passed to EditScript(...), but it must be that unique final-path resource and must not be the old closed Script instance.
- Retains old ScriptEditorBase/TextEdit instance IDs for diagnostics only. Reuse of either editor control is not itself a failure when the unique final Script/path relationship, old-path rejection, and captured buffer text are safe.
- Restores caret, horizontal/vertical scroll, and selection only after deferred Script/TextEdit verification, then requests restoration of the previously active script when the renamed target was not previously active.
- Mutates _systems only after verified final filesystem success, updates every matching stored script path case-insensitively to the exact final casing, preserves folder, linked-scene, lock and ordering data, saves only after an actual entry update, rebuilds with preserved expansion, and reselects the exact originating system/folder/entry including filtered results.
- Restores System Explorer tree selection before reopen without taking final focus, then restores selection and tree focus again after editor success or failure; only that terminal path ends ScriptEditorSync suppression. A successful filesystem rename is never rolled back because of a later editor-state failure.
- Does not use Resource.TakeOverPath(...), does not keep a parallel resource-rebind strategy, does not write Godot's .godot editor cache files, and does not repair already-existing duplicate tabs automatically.

SystemExplorerPlugin.EntryHelpers.cs
- Contains shared helpers for selection lookup, entry mutation, metadata parsing, path/folder utilities, display names, and entry normalization.
- Contains helpers for script entries, scene entries, linked scene metadata, folder entries, lock markers, and selected filter result updates.

SystemExplorerPlugin.Persistence.cs

- Provides EnsureSystemsLoadedForTreeOperation(...), EnsureSystemAvailable(...), EnsureSystemsAvailable(...), disk recovery, serialization, and deserialization.

SystemExplorerPlugin.Settings.cs

- Contains settings-backed toggles for context menu icons and Quick Actions.

SystemExplorerPlugin.QuickAction.Beautify.cs
- Handles the Quick Action flow for Beautify Script / Beautify Scripts.
- Uses ScriptPathUtility.Normalize(...) for general script and folder path string normalization while retaining .cs filtering, folder Trim('/'), case-insensitive collection policies, and operation ordering at their existing call sites.
- Collects and validates single-script, active-editor script, and batch targets; stores pending beautify actions when CSharpier needs installation; and routes formatting through the shared CSharpier helpers.
- Handles beautify result summaries, unchanged/skipped/failed handling, safe write checks, and main operation orchestration.
- Uses ScriptResourceRefreshService at the existing unchanged-after-autosave, failed-write rollback, and successful-write refresh points.
- Preserves refresh ordering relative to rollback, committed open-buffer application, and editor view-state restoration.
- Uses ScriptEditorSync batch context preservation for Beautify Scripts so batch formatting does not trigger post-operation active-script selection sync.
- Delegates editor view-state capture/restore and open editor-buffer validation/application helpers to SystemExplorerPlugin.QuickAction.Beautify.EditorState.cs.

SystemExplorerPlugin.QuickAction.Beautify.EditorState.cs
- Contains Beautify-specific editor-state and open editor-buffer safeguards.
- Captures/restores TextEdit caret, first visible line, horizontal/vertical scroll, and focus state for single-script Beautify.
- Handles deferred restore passes so Godot does not unnecessarily throw the user to the top of the file after formatting or refresh.
- Retains Beautify autosave I/O, exact text verification, and the preliminary editor-first text assignment performed before disk writes.

## Editor integration classes

EditorIntegration/ScriptEditing/OpenScriptEditorBuffer.cs
- Defines the internal immutable buffer reference model under SystemExplorer.EditorIntegration.ScriptEditing.
- Stores an already-normalized script path together with its matched Godot TextEdit instance.
- Does not normalize paths, perform matching, access files, or orchestrate editor operations.

EditorIntegration/ScriptEditing/ScriptPathUtility.cs
- Defines the internal stateless ScriptPathUtility under SystemExplorer.EditorIntegration.ScriptEditing.
- Owns general script-path string normalization by trimming surrounding whitespace, replacing backslashes with forward slashes, and returning an empty string for null.
- Preserves casing and does not localize or globalize paths.
- Does not validate res://, .cs, or file existence and does not deduplicate collections.
- Has no Godot, filesystem, editor-buffer, Refactor Namespace, Beautify, CSharpier, tree, or operation-specific dependencies.

EditorIntegration/ScriptEditing/ScriptEditorBufferLookupResult.cs
- Defines the small non-activating lookup result model and failure enum under SystemExplorer.EditorIntegration.ScriptEditing.
- Carries matched buffers, unsafe text-match paths, unmatched required paths, and structured failure information without embedding Refactor Namespace-specific warning text.
- Preserves the existing case-insensitive buffer dictionary shape used by compatibility wrappers.

EditorIntegration/ScriptEditing/ScriptEditorTabService.cs
- Defines the internal Godot 4.6-specific Script Editor tab lifecycle adapter under SystemExplorer.EditorIntegration.ScriptEditing.
- Validates the exact active Script, ScriptEditorBase, and TextEdit instance before locating the unique Script Editor File menu by the verified Godot 4.6 menu-ID signature.
- Invokes only Godot's own FILE_MENU_CLOSE route and verifies through ScriptEditor.GetOpenScripts() that the exact old Script instance and case-insensitive old path are no longer open.
- Returns an explicit success/failure result and safely aborts when the target editor pair, File menu, close command, or close verification is ambiguous or unavailable.
- Verifies only the live Script Editor state; persistence and pruning of closed-script sections in Godot's internal cache remain Godot-owned and are never manipulated by the service.
- Does not rename files, load/rewrite script text, own autosave policy, mutate _systems/JSON/tree state, scan the filesystem, write .godot cache files, or know about Refactor Namespace, Beautify, CSharpier, or delete/remove workflows.

EditorIntegration/ScriptEditing/ScriptEditorBufferStateService.cs
- Defines the internal stateless TextEdit buffer-state service under SystemExplorer.EditorIntegration.ScriptEditing.
- Owns saved/unsaved checks through GetVersion()/GetSavedVersion() and collects unsaved OpenScriptEditorBuffer paths with case-insensitive deduplication while preserving first-occurrence order.
- Owns generic TagSavedVersion() transitions after successful autosaves.
- Applies already-committed text to an identified TextEdit, assigning only when text differs, then clearing undo history before marking the current version saved even when the text was already identical.
- Does not read/write files, normalize or match paths, iterate operation-specific dictionaries, activate scripts, refresh Godot resources, manage view state, show messages, or know about Refactor Namespace, Beautify, CSharpier, the tree, or _systems.

EditorIntegration/ScriptEditing/ScriptEditorBufferAutosaveService.cs
- Defines the internal per-buffer autosave service plus its structured result and failure enum under SystemExplorer.EditorIntegration.ScriptEditing.
- Owns generic saved-buffer disk verification, unsaved TextEdit writes to the existing script path, normalized read-back verification, and the saved-version transition only after successful write and verification.
- Returns whether the operation succeeded, whether an autosave occurred, the affected script path, and a structured failure kind without embedding operation-specific user messages.
- Receives general read, write, and disk-verification comparison delegates through constructor injection; the plugin supplies ScriptTextFileService.ReadText, WriteText, and TextsMatchForDiskVerification directly.
- Uses ScriptEditorBufferStateService only for saved/unsaved detection and MarkCurrentVersionSaved(...).
- Does not know about Refactor Namespace, Beautify, rename/remove, editor activation/focus, rollback policy, resource refresh, tree state, _systems, or JSON persistence.

EditorIntegration/ScriptEditing/ScriptEditorBufferLocator.cs
- Owns the non-activating open-buffer matching previously implemented in SystemExplorerPlugin.ScriptEditorBuffers.cs.
- Preserves current-editor-first matching, original/updated text matching, unique-text ambiguity checks, used-TextEdit protection, the existing indexed GetOpenScripts()/GetOpenScriptEditors() relation, saved-buffer disk verification, and the disk-text fallback when list counts differ.
- Delegates only its saved/unsaved TextEdit check to ScriptEditorBufferStateService; its matching algorithm, lookup priority, path behavior, and disk-verification flow are otherwise unchanged.
- Receives path normalization and file-existence behavior through injected delegates; the plugin supplies ScriptPathUtility.Normalize for path normalization, while disk-text reading and disk-verification comparison delegates are supplied directly by ScriptTextFileService.
- Does not activate scripts, call EditorInterface.EditScript(...), autosave or write files, apply text, change editor focus/view state, manage ScriptEditorSync or batch context, inspect namespaces/usings, show UI, or mutate the System Explorer tree.

EditorIntegration/ScriptEditing/ScriptTextFileService.cs
- Defines the internal stateless text-file service under SystemExplorer.EditorIntegration.ScriptEditing.
- Owns safe general text-file reading and writing, including res:// and user:// globalization before System.IO access.
- Owns UTF-8 writes without BOM and preserves the existing empty-string/false fallback behavior for missing files and caught I/O failures.
- Owns CRLF/standalone-CR to LF normalization and ordinal disk-text comparison after that normalization.
- Does not normalize general script paths, create directories, refresh Godot resources, autosave or update editor buffers, display UI, log, or hold mutable operation state.

EditorIntegration/ScriptEditing/ScriptResourceRefreshService.cs
- Defines the internal stateless Godot editor-integration service under SystemExplorer.EditorIntegration.ScriptEditing.
- Owns synchronous refresh of script resource paths after their text has already changed.
- Filters blank paths, normalizes with ScriptPathUtility.Normalize(...), and deduplicates case-insensitively while preserving the first occurrence order.
- Materializes the normalized unique paths before retrieving EditorFileSystem through EditorInterface.Singleton.
- Calls EditorFileSystem.UpdateFile(...) synchronously once for each unique path and returns without side effects when EditorFileSystem is unavailable.
- Does not read or write files, apply text to editor buffers, handle deferred buffer refresh, call EditorFileSystem.Scan(), load resources, show UI, or log.
- Does not know about Beautify, Refactor Namespace, CSharpier, the tree, _systems, or JSON persistence.



## RefactorNamespace classes

QuickActions/RefactorNamespace/NamespaceTextRewriter.cs
- Defines the standalone internal static class SystemExplorer.QuickActions.RefactorNamespace.NamespaceTextRewriter.
- Owns pure C# text logic for namespace name validation, namespace detection, namespace declaration replacement, adding namespace blocks around scripts without namespaces, replacing matching using directives, and adding a normal using directive after a matching existing using without duplicates.
- Privately owns insertion-point calculation after top using/global using directives and namespace-body indentation.
- Has no filesystem, Godot editor-state, dialog, batch orchestration, tree, persistence, or SystemExplorerPlugin dependency.


QuickActions/RefactorNamespace/NamespaceScriptSnapshot.cs
- Defines the immutable, Godot-independent representation of an already-read C# script under SystemExplorer.QuickActions.RefactorNamespace.
- Stores the normalized script path and the script text captured at snapshot time through get-only properties.
- Has no Godot, editor, filesystem, mutation, or behavior dependencies.


QuickActions/RefactorNamespace/NamespaceRefactorScopeResolver.cs
- Defines the Godot-independent NamespaceRefactorScopeResolver under SystemExplorer.QuickActions.RefactorNamespace.
- Builds normalized, case-insensitively deduplicated C# script-path scopes while preserving the first occurrence order within each supplied group.
- Combines already-normalized or raw path groups into one ordered, case-insensitively deduplicated load scope used by NamespaceRefactorPreparationService.
- Filters System and Folder targets from neutral NamespaceRefactorScriptEntry objects; exact folder matches and descendants are included without matching siblings or text-prefix-only folder names.
- Builds recursive project-wide C# inventory through System.IO.Directory and combines scanned paths with normalized linked-script fallback paths.
- Reports project scan failures through NamespaceRefactorProjectInventoryResult without Godot warnings, dialogs, or debug logging.
- NamespaceRefactorScriptEntry is an immutable neutral model containing only ScriptPath and FolderPath; it has no metadata, system, lock, scene, plugin, Godot, or mutable collection dependency.
- NamespaceRefactorProjectInventoryResult exposes the resolved script paths, an optional failure message, and whether fallback was used.
- Receives only path normalization/globalization/localization delegates; the plugin supplies ScriptPathUtility.Normalize for general normalization, and the resolver does not access _systems directly or cache operation paths.
- Does not read script text, load snapshots, build namespace plans, manage editor buffers, write files, or apply changes.

QuickActions/RefactorNamespace/NamespaceRefactorSnapshotLoader.cs
- Defines the Godot-independent NamespaceRefactorSnapshotLoader and NamespaceSnapshotLoadResult under SystemExplorer.QuickActions.RefactorNamespace.
- Reads already-decided C# script paths into immutable NamespaceScriptSnapshot objects through small normalization, existence-check, and text-read delegates supplied by the plugin; the normalization delegate is ScriptPathUtility.Normalize.
- Filters blank/non-C# paths, normalizes and deduplicates paths case-insensitively, and builds a read-only snapshot index using StringComparer.OrdinalIgnoreCase.
- Preserves the first-request order separately for target-sensitive plan input and can reuse an existing snapshot index through LoadInto(...) without rereading paths.
- When existing snapshots are reused, normalizes each snapshot's own Path as its identity, recreates only snapshots whose stored path is not already normalized, and prevents an external dictionary key from creating an alias or disagreeing with the indexed snapshot path.
- If multiple imported snapshots normalize to the same path, the later enumerated snapshot retains the existing dictionary assignment behavior and wins.
- Reports missing paths and delegate failures as separate read-only lists without Godot warnings, dialogs, or debug logging.
- Receives one combined operation path list from NamespaceRefactorPreparationService, so a path shared by target/reference/declaration roles is read at most once within that preparation.
- Does not decide System/Folder/single scope, inspect _systems, assign operational snapshot roles, build namespace-plan semantics, write files, cache snapshots across operations, or handle editor buffers/state.

QuickActions/RefactorNamespace/NamespaceRefactorPreparationService.cs
- Defines the Godot-independent NamespaceRefactorPreparationService plus immutable NamespaceRefactorPreparationResult and NamespaceRefactorNamespaceScanResult models under SystemExplorer.QuickActions.RefactorNamespace.
- Accepts already-decided target, using-reference, and namespace-declaration path roles from the plugin rather than interpreting metadata or systems data.
- Normalizes and case-insensitively deduplicates each path role separately through NamespaceRefactorScopeResolver while preserving the first occurrence order, especially target order.
- Combines all roles into one load scope, loads them through NamespaceRefactorSnapshotLoader, and maps the same indexed NamespaceScriptSnapshot instances back into each operational role without rereading shared paths.
- Delegates replace and add plan semantics directly to NamespaceRefactorPlanBuilder without parsing declarations, deciding full/partial moves, rewriting usings, or constructing pending-write text itself.
- Scans batch-dialog namespaces from loaded snapshots through NamespaceTextRewriter.GetNamespaceFromText(...), returning a case-insensitive read-only namespace dictionary plus missing and failed paths.
- NamespaceRefactorPreparationResult exposes the plan result, shared snapshot-load result, normalized role paths, and target-filtered missing/failed path lists needed by existing plugin diagnostics.
- NamespaceRefactorNamespaceScanResult defensively exposes namespace, missing-path, and failed-path collections without logging or UI behavior.
- Does not reference SystemExplorerPlugin, _systems, Tree/TreeItem, editor controls, Godot APIs, FileAccess, editor buffers, dialogs, warnings, debug logging, file writes, or plan application.

QuickActions/RefactorNamespace/NamespaceRefactorPlanBuilder.cs
- Defines the Godot-independent NamespaceRefactorPlanBuilder and its immutable plan/result models under SystemExplorer.QuickActions.RefactorNamespace.
- NamespaceRefactorPlan contains the selected script path plus read-only original-text and final pending-write dictionaries using case-insensitive path comparison.
- NamespaceRefactorPlanResult and NamespaceRefactorPlanFailure provide the completed plan plus the limited skip/failure information needed by existing plugin warnings and debug logging.
- Builds side-effect-free replace-namespace plans shared by single-script and batch flows, including target declaration rewrites and matching using-directive updates through NamespaceTextRewriter.
- Receives already-role-mapped snapshots from NamespaceRefactorPreparationService and keeps target scripts, using-reference candidates, and namespace declaration inventory semantically separate.
- Determines whether the old namespace remains after the planned target rewrites, including failed or non-matching targets and unselected project scripts.
- Replaces old using directives during a full namespace move; during a partial move it preserves the old using and adds the new using without duplication.
- Builds add-namespace plans for scripts without namespaces through NamespaceTextRewriter.
- Preserves original source text once per path while producing one final pending write after all applicable namespace and using changes.
- Does not normalize paths, read or write files, use Godot, manage dialogs/editor state, or reference SystemExplorerPlugin.
- The completed responsibility chain is plugin metadata and systems data -> NamespaceRefactorScopeResolver -> editor save-first preflight in the plugin -> NamespaceRefactorPreparationService -> NamespaceRefactorSnapshotLoader and NamespaceRefactorPlanBuilder -> safe apply/editor integration in SystemExplorerPlugin.
